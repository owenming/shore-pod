# 上岸舱·小工具小程序 第一版产品原型

> 版本：V1.0 MVP  
> 产品形态：微信小程序  
> 目标：3 个月内完成冷启动验证，复制粉笔、背书匠、华图、半月谈、中公五家 APP 的核心亮点，以「轻工具 + 私域转化」切入考公/考编市场。

---

## 一、产品定位

### 1.1 产品名称

**上岸舱**

### 1.2 一句话定位

> 考公/考编人的碎片时间提分舱——每天 5 分钟，时政、申论、行测一起练。

### 1.3 核心场景

| 场景 | 用户状态 | 需求 | 对应功能 |
|------|----------|------|----------|
| 早晨通勤/睡前 | 在职考生，碎片时间 | 快速积累时政 | 每日时政一测 |
| 背诵记忆瓶颈 | 申论素材记不住 | 关键词挖空、反复记忆 | 申论挖空背诵 |
| 做题前热身 | 刷题前激活知识点 | 行测公式/口诀速记 | 行测速记卡 |
| 考前冲刺 | 需要反馈和方向 | 薄弱点诊断、学习报告 | 学习数据看板 |

### 1.4 目标用户（第一版聚焦）

- **主力**：25-30 岁在职考公/考编人
- **次要**：应届备考学生
- **核心特征**：时间不固定、愿意为高效提分付费、习惯在微信生态内学习

### 1.5 产品边界

- **做轻**：不做系统直播课、不做重社区、不做人工答疑
- **做高频**：每天来一次，打卡有闭环
- **做转化**：免费工具 → 低价会员 → 私域课程

## 二、竞品亮点复制清单（第一版侧重）

| 来源竞品 | 可复制的亮点 | 在上岸舱中的落地 | 第一版是否实现 | 优先级 |
|----------|-------------|------------------|----------------|--------|
| **半月谈** | 时政每日测验（高频刚需） | 首页首屏「每日时政一测」，每天 5 题，做完可看排名/解析 | ✅ 是 | P0 |
| **半月谈** | 督学返学费机制 | 7 天时政打卡挑战，全勤返¥5/¥9.9（抵会员费），提升打开率 | ✅ 是 | P0 |
| **背书匠** | AI 关键词挖空 + 艾宾浩斯复习 | 「申论挖空背诵」模块：导入金句 → AI 自动挖空 → 朗读+背诵 → 按遗忘曲线提醒 | ✅ 是 | P0 |
| **背书匠** | 资料广场（UGC） | 「素材库」：内置申论金句、时政要点、行测公式；后续支持用户上传 | ⚠️ 部分 | P1 |
| **粉笔** | AI 批改 + 薄弱点诊断 | 轻量版「申论小题批改」：输入 200 字答案，AI 给出评分+改进建议 | ✅ 是 | P1 |
| **粉笔** | 智能题库 + 错题本 | 行测速记卡 + 时政错题本，支持收藏和重复练习 | ✅ 是 | P1 |
| **华图** | 分省模考/省情 | 首页可切换「国考/省考/事业单位」，每日时政题目按省份/岗位差异化 | ⚠️ 部分 | P1 |
| **华图** | 口诀歌速记 | 行测速记卡中内置「数量关系秒杀口诀」「图形推理口诀」等 | ✅ 是 | P1 |
| **中公** | 免费题库引流 | 首页行测速记卡和时政题库全部免费，降低进入门槛 | ✅ 是 | P0 |
| **中公** | 学习报告/成长方案 | 「我的」页面展示学习数据、薄弱点、连续打卡天数、建议学习计划 | ✅ 是 | P1 |

### 2.1 第一版侧重的三大亮点

1. **半月谈式时政每日测验**（P0）：这是最高频、最合规、最裂变的入口。每天 5 道题，做完自动生成「今日战报」卡片，用户可分享到群/朋友圈，自然带来新用户。
2. **背书匠式 AI 挖空背诵**（P0）：解决申论提分最大痛点——素材记不住。通过 AI 自动挖空，把长段落变成可背诵的卡片，形成强粘性。
3. **中公式免费题库 + 学习报告**（P1）：通过免费行测速记卡和时政题库建立信任，再用数据报告引导用户认识到自己的不足，自然过渡到付费会员。

**交互说明**

| 元素 | 交互 | 说明 |
|------|------|------|
| 省份切换 | 点击弹出选择器，可选「国考/通用+各省」 | 用于后续分省时政题目 |

- 未登录：点击答题时先请求微信授权
- 未报名打卡：7 天打卡按钮显示「¥9.9 报名挑战」
- 已报名未打卡：显示「今日未打卡，去完成」
- 已报名已打卡：显示「今日已打卡，明天继续」
- 会员用户：去除限制提示，显示会员标识

**布局结构**

```

│                               │
│  [解析]                       │
│  本题考查...原文出自...       │
├─────────────────────────────┤
│  [下一题]  /  [交卷看战报]    │

```

**交互说明**

| 场景 | 交互 | 说明 |
|------|------|------|
| 选择答案 | 选项高亮，自动显示解析 | 即时反馈，增强学习效果 |
| 查看解析 | 默认答完即展示，可展开/收起 | 解析需标注出处（如：学习强国、新华社） |
| 交卷 | 答完第 5 题后展示「今日战报」 | 战报包含：正确率、击败比例、今日考点 |
| 分享 | 战报页生成海报，可保存/分享到朋友圈 | 分享语：「今日时政 5 题，我答对了 X 题，你也来试试」 |
| 错题 | 自动加入错题本，可收藏 | 便于后续复习 |

**功能细节**

- 每日 0 点刷新题库，保持新鲜感
- 每题限时 60 秒（可选），增加紧迫感
- 答错后显示「相关知识点卡片」，可跳转速记卡

```
    ↓

    ↓

```

## 七、技术实现建议

### 7.1 技术栈

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 前端 | 微信小程序原生 / Taro | 推荐 Taro，方便后续转 APP |
| 后端 | Node.js + Express / 云开发 | 初期可用微信云开发，降低成本 |
| 数据库 | MongoDB / 云开发数据库 | 题目、素材、用户数据 |
| AI 能力 | DeepSeek / 文心一言 / Kimi API | 用于挖空、批改、出题 |
| 语音 | 微信小程序自带 TTS | 朗读背诵内容 |
| 支付 | 微信支付 | 会员/挑战付费 |
| 消息推送 | 微信小程序订阅消息 | 复习提醒、打卡提醒 |

### 7.2 AI 能力调用规划

| 功能 | AI 调用 | 输入 | 输出 |
|------|---------|------|------|
| 申论挖空 | DeepSeek/Kimi | 申论素材原文 | 挖空后的文本 + 挖空关键词列表 |
| 申论小题批改 | DeepSeek/Kimi | 题目 + 用户答案 | 评分 + 改进建议 + 范文参考 |
| 时政题目生成 | 内部题库 + AI 生成 | 时政新闻原文 | 单选题 + 解析 + 出处 |
| 学习报告 | 规则 + 轻量 AI | 用户答题数据 | 薄弱点分析 + 学习建议 |

### 7.3 小程序特殊注意事项

1. **分享合规**：分享海报必须包含小程序码，避免诱导分享被处罚
2. **订阅消息**：每次提醒需用户主动触发订阅，不能滥用
3. **虚拟支付**：iOS 端虚拟支付受限，需做引导或走公众号/H5 支付
4. **内容合规**：时政内容必须引用官方信源，不原创评论，不涉敏感
5. **授权规范**：用户拒绝授权后仍可继续使用，不得强制

## 十、总结

上岸舱第一版小程序的核心策略是：**用「时政每日一测」做高频入口，用「申论挖空背诵」做深度留存，用「7 天打卡挑战」做私域转化，用「免费题库+学习报告」建立信任。**

复制五家竞品的亮点但不做重资产：
- 学半月谈：做时政每日测验 + 返学费打卡
- 学背书匠：做 AI 挖空 + 艾宾浩斯复习
- 学粉笔：做轻量 AI 批改 + 数据驱动的学习报告
- 学华图：做分省/模块化的速记内容
- 学中公：用免费内容引流，用会员和私域变现

这个产品原型可在 6-8 周内完成开发，适合小团队冷启动验证。
